# 5. Navigation Flow

## Primary Navigation Structure

### Customer Experience
- Home
  - Landing / Explore
  - Login
  - Register
  - Forgot Password
  - Dashboard
    - Search Stations
    - Station Details
    - Booking Flow
    - Booking Summary
    - Payment
    - Payment Success
    - Booking History
    - Favourite Stations
    - Profile
    - Notifications
    - Settings

### Admin Experience
- Admin Login
  - Admin Dashboard
    - Manage Stations
    - Manage Chargers
    - Manage Users
    - Manage Bookings
    - Revenue Analytics
    - Settings

## Navigation Principles

- Keep primary actions visible and consistent
- Use a minimal number of steps between entry and completion
- Support deep linking to booking and account areas
- Ensure admin and user navigation remain clearly separated

## Suggested IA

### User Top Navigation
- Explore
- Dashboard
- Bookings
- Notifications
- Profile

### Admin Sidebar Navigation
- Overview
- Stations
- Chargers
- Users
- Bookings
- Analytics
- Settings

## Flow Summary

A user can move from landing page to sign in, search stations, choose a slot, confirm a booking, pay, and view confirmation without confusion. Each section should preserve context and offer clear return paths.
