# 1. Design System

## 1. Brand Identity

### Brand Positioning
VoltGo should be perceived as a premium, intelligent, and dependable EV charging platform. The brand tone should be calm, modern, and reassuring, reflecting both technology and convenience.

### Brand Personality
- Confident
- Minimal
- Smart
- Trustworthy
- Premium
- Efficient

### Brand Messaging
- Charge smarter
- Reserve with confidence
- Drive without friction

## 2. Logo Concept

### Concept Direction
A modern wordmark paired with a simple abstract charging bolt or circular energy symbol. The logo should feel clean and technology-driven, with a premium and scalable form.

### Logo Requirements
- Works well in light and dark backgrounds
- Readable at small sizes
- Suitable for app icon and web favicon
- Should not feel overly complex or ornamental

## 4. Typography

### Primary Type Family
- Inter, SF Pro Display, or system sans for a sleek modern feel

### Type Scale
- Display: 40px / 700
- H1: 32px / 700
- H2: 24px / 600
- H3: 20px / 600
- Body Large: 16px / 400
- Body: 14px / 400
- Caption: 12px / 400

### Usage Guidelines
- Headings should stay crisp and minimal
- Body text should prioritise rhythm and readability
- Important UI text should use higher contrast

## 5. Icon Style

### Style Direction
- Rounded outline icons for a polished SaaS feel
- Minimal stroke weight
- Consistent 24px grid
- Use simple, familiar EV and mobility metaphors

### Icon Categories
- Navigation icons
- Station status icons
- Payment and booking icons
- Security and verification icons

## 6. Spacing System

Use an 8-point spacing scale.

- 4px
- 8px
- 12px
- 16px
- 24px
- 32px
- 40px
- 48px
- 64px
- 80px

## 7. Border Radius

- Small: 8px
- Medium: 12px
- Large: 16px
- Pill: 999px

## 8. Shadows

### Shadow Levels
- Shadow 1: subtle elevation for cards
- Shadow 2: medium elevation for floating panels
- Shadow 3: strong elevation for modals and overlays

## 9. Elevation Levels

- Level 0: flat surfaces
- Level 1: cards and containers
- Level 2: popovers and dropdowns
- Level 3: modals and dialogs

## 10. Button Variants

### Primary Button
- Solid filled action button for main tasks

### Secondary Button
- Neutral outline for supporting actions

### Ghost Button
- Low-emphasis action for lightweight interactions

### Destructive Button
- Red-based style for delete, cancel, or warning actions

### Icon Button
- Small circular action button for compact controls

## 11. Input Fields

### Field Style
- Rounded corners
- Clear labels
- Focus states with accent border and subtle glow
- Helpful helper text and validation messages

### Input States
- Default
- Focus
- Error
- Success
- Disabled

## 12. Cards

### Card Principles
- Rounded corners
- Light borders
- Soft shadow
- Clean spacing
- High clarity for station and booking information

## 13. Tables

### Table Guidelines
- Clear headers
- Horizontally aligned content
- Sticky headers in admin views
- Row hover states for better scanning

## 14. Badges

Use badges for statuses such as:
- Available
- Booked
- Confirmed
- Payment Pending
- Maintenance

## 15. Chips

Chips should be used for:
- Filter categories
- Quick selection options
- Station attributes such as fast charging or accessible charging

## 16. Toast Notifications

### Toast Behaviour
- Short, non-blocking messages
- Positioned top-right or top-center
- Use success, error, warning, and info variants

## 17. Loading States

- Skeleton layout for cards and forms
- Progress indicators for booking and payment processes
- Subtle transitions rather than abrupt changes

## 18. Empty States

Empty states should feel intentional and helpful, not blank or broken.

### Examples
- No bookings yet
- No saved stations
- No search results found

## 19. Skeleton Screens

Use skeletons for:
- Station cards
- Booking summary sections
- Dashboard widgets
- Admin tables

## 20. Modal Guidelines

- Modal should support a single focused task
- use clear title, short body, and strong CTA actions
- close action should be obvious
- avoid overuse for routine operations
