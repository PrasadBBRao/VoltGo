# 9. Coding Conventions

## General Rules

- Use clear and descriptive names
- Keep functions small and focused
- Prefer modularity over duplication
- Follow consistent file naming conventions
- Document non-obvious logic

## Frontend Conventions

- Use functional components and hooks
- Keep components small and reusable
- Use PascalCase for component names
- Use camelCase for variables and functions
- Use lowercase kebab-case for file names

## Backend Conventions

- Use snake_case for Python variables and functions
- Use uppercase for constants
- Prefer service-based logic over inline business rules
- Keep routes thin and delegate to services

## Formatting Rules

- Follow Prettier formatting rules
- Use ESLint to maintain code quality
- Keep import groups organized

## Why This Matters

Consistent conventions improve readability, onboarding speed, and long-term maintainability.
