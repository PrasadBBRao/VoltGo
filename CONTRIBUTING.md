# Contributing to VoltGo

## Contribution Guidelines

1. Create a feature branch from the latest main branch.
2. Keep changes focused and well-documented.
3. Do not implement application features unless the task explicitly requires them.
4. Follow linting, formatting, and naming conventions.
5. Update documentation when changing architecture or setup behavior.

## Branching Strategy

- main: production-ready code
- develop: integration branch for ongoing work
- feature/*: new feature work
- bugfix/*: issue fixes
- hotfix/*: urgent production fixes

## Code Review Expectations

- Keep pull requests small and reviewable
- Include a clear summary of changes
- Add or update documentation when needed
